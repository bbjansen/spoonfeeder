# Getting Started

This guide covers installation, configuration, and running a project generated by spoonfeeder.

---

## Prerequisites

- **Node.js** 22+ (see `.nvmrc`)
- **pnpm** 10+

## Installation

```bash
# Clone the repository
git clone <repo-url>
cd <project-name>

# Install dependencies (also installs Git hooks)
pnpm install

# Copy environment variables
cp .env.example .env
```

## Environment Variables

| Variable | Default | Description                     |
| -------- | ------- | ------------------------------- |
| `PORT`   | `3000`  | HTTP port the server listens on |

## Development

```bash
pnpm start:dev       # Start with file watching
pnpm start:debug     # Start with debugger attached
pnpm build           # Compile to dist/
pnpm start:prod      # Run compiled output
```

## Testing

Tests live in the `tests/` directory at the project root, mirroring the `src/` structure.

```bash
pnpm test            # Run unit tests
pnpm test:unit       # Run unit tests (explicit)
pnpm test:integration # Run integration tests (Testcontainers)
pnpm test:e2e        # Run end-to-end tests
pnpm test:all        # Run all test suites
pnpm test:watch      # Run unit tests in watch mode
pnpm test:cov        # Run unit + integration tests with coverage
```

## Code Quality

```bash
pnpm lint            # Lint and auto-fix
pnpm format          # Format with Prettier
```

Linting and formatting run automatically on staged files via a pre-commit hook. Unit tests and type checking run on pre-push.

## Project Structure

```
src/
  main.ts                   # Application bootstrap
  app.module.ts             # Root module
  config/                   # Configuration factories
  shared/                   # Cross-cutting concerns
    constants/
    decorators/
    filters/                # GlobalExceptionFilter
    guards/
    interceptors/           # ResponseInterceptor
    pipes/
    utils/
  infrastructure/           # External integrations
    database/
      entities/             # TypeORM entities
      migrations/
      repositories/         # Custom repositories
    http/                   # External HTTP API clients
    queue/                  # Message queue (RabbitMQ)
    storage/                # File / S3 storage
    notifications/          # Slack, email, etc.
  app/
    modules/                # Feature modules

tests/
  unit/                     # Unit tests (mirrors src/)
  integration/              # Integration tests (Testcontainers)
  e2e/                      # End-to-end tests
  factories/                # Shared test data factories
  helpers/                  # Shared test utilities
```

Path alias `@/*` resolves to `src/*`.
